O Futuro Brilhante do io.js

Um Roadmap Orientado à Comunidade

Para a comunidade, pela comunidade

O io.js é um substituto transparente para o node.js e é compatível com quase todos os módulos no npm.

O io.js é um projeto código livre orientado à comunidade. Foi criado pelos principais contribuidores do node.js, mas nos primeiros meses atraíu mais programadores ativos do que o node.js alguma vez teve em toda a sua história.

O que é que as pessoas querem do io.js?

  1. Estabilidade
  2. Transparência (Debugging e Tracing)
  3. Melhores Streams
  4. Suporte a Longo Prazo
  5. Adoção de Standards em Desenvolvimento
  6. Localização

Lista de interesses compilada do projeto roadmap. Contribuidores são bem-vindos!

Política de Estabilidade

Temos o maior ecossistema de pacotes de todas as linguagens. Não podemos perder isso.

  • Não removemos APIs JS
  • Versionamento Semântico
    • Qualquer adição à API implica um incremento à versão menor.
    • Mudanças na API C++ do V8 tratadas pelo nan implicam um incremento da versão menor e um incremento da versão maior caso não seja possível.

Da secção de Política de Estabalidade do Roadmap. Contribuidores são bem-vindos.

NG (Next Generation)

  • Uma nova plataforma para Aplicações Javascript modernas.
  • Permanecerá retrocompatível indefinidamente.
  • Novas APIs podem ser focadas em ES6/7 sem quebrar módulos e APIs antigos.
  • Separa o trabalho em APIs futuras da plataforma atual.

Discussões NG estão a decorrer no repositório NG.

Debugging & Tracing

O núcleo deve possibilitar um ecossistema de ferramentas de debugging e tracing, em vez de encorajar abstrações de alto nível como os domains.

  • Integrar com o trace_event (Nova API de trace para o V8 e Chrome).
  • Documentar e evangelizar um ecossistema de debugging.
  • Unificar o endpoint de tracing de sistema (DTrace, LTTng, etc).

Trabalho a ser liderado pelo Tracing Working Group. Contribuidores são bem-vindos!

Streams

  • O Streams Working Group assumiu o readable-stream, que é agora o módulo oficial de stream do io.js.
  • Corrigir todos os problemas de compatibilidade existentes.
  • Simplificar a criação de streams para evitar erros de utilizador.
  • Explorar a interface WHATWG Streams e identificar problemas de compatibilidade.
  • Melhorar a performance das streams.

Trabalho a ser lidarado pelo Streams Working Group. Contribuidores são bem-vindos!

Suporte a Longo Prazo

  • Novos lançamentos vão incluir dependências estáveis e suportadas.
  • Lançamentos de correções de versões antigas serão publicadas enquanto houver pessoas a contribuir ativamente.
  • O io.js irá assumir responsabilidade pelas versões antigas das dependências como o V8 que aparecem em versões anteriores quando os seus contribuidores deixam de as suportar.

Resumo da Política de Estabilidade. Contribuidores são bem-vindos!

Localização

  • Reduzir o impacto de incluir o icu (footprint binário e de memória).
  • Incentivar a uma comunidade de localização vibrante.
    • Comunidades autónomas de idiomas (+30 registadas até agora)
    • Esforços de evangelização para crescer a filiação (+160 membros até agora)
    • Melhorar as ferramentas para traduzir website, documentação da API, instaladores e slides.

Podes envolver-te juntando-te à tua comunidade do teu idioma :)
