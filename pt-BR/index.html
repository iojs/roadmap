O Futuro Brilhante do io.js

Um Roadmap Guiado pela Comunidade

Para a comunidade, pela comunidade

O io.js é um substituto imperceptível para o node.js e é compatível com praticamente todos os módulos no npm.

O io.js é um projeto open source guiado pela comunidade. Ele começou através dos principais contribuidores do node.js, mas nos primeiros meses já atraiu mais desenvolvedores ativos que o projeto node.js teve em toda a sua história.

O que as pessoas querem do io.js?

  1. Estabilidade
  2. Transparência (Debug e Tracing)
  3. Melhores Streams
  4. Suporte de Longo Prazo
  5. Abraçar padrões em desenvolvimento
  6. Localização

Lista de preocupações compilada do projeto roadmap. Contribuidores bem vindos!

Política de Estabilidade

Nós temos o maior ecossistema de pacotes de qualquer linguagem, não podemos perder isso.

  • Não removemos APIs JS.
  • Versionamento Semântico
    • Qualquer adição de API significa um incremento da versão menor.
    • Alterações na API C++ do V8 tratadas pelo nan significam um incremento na versão menor e um incremento maior em caso contrário.

NG (Next Generation)

  • Uma nova plataforma para aplicações JavaScript modernas.
  • Manter-se-á retrocompatível indefinidamente.
  • Novas APIs podem ser voltadas para ES6/7 sem quebrar antigos módulos e APIs.
  • Separa trabalho em futuras APIs de melhorias na plataforma atual.

Discussões NG estão ocorrendo no repositório NG.

Depuração & Rastreamento

O núcleo deve permitir um grande e diversificado ecossistema de ferramentas de depuração e rastreamento ao invés de endossar abstrações de alto nível como domains.

  • Integra com trace_event (Nova API trace para o V8 & Chrome).
  • Documenta e evangeliza um ecossistema de depuração.
  • Unifica endpoints de sistema de rastreamento (DTrace, LTTng, etc)

Trabalho liderado pelo Tracing Working Group. Contribuidores bem vindos!

Streams

  • Streams Working Group assumiu o readable-stream, que agora é o módulo stream oficial do io.js.
  • Corrigir todos problemas de compatibilidade existentes.
  • Simplificar criação de streams para evitar erro do usuário.
  • Explorar a interface WHATWG Streams e identifica problemas de compatibilidade.
  • Melhorar performance de streams.

Trabalho liderado pelo Streams Working Group. Contribuidores bem vindos!

Suporte de Longo Prazo

  • Novas versões de frente chegarão com dependências estáveis e suportadas.
  • Correções de versões anteriores serão publicadas enquanto as pessoas estiverem ativamente contribuindo.
  • io.js se responsabilizará por versões antigas de dependências, como o V8, que aparecem em versões anteriores quando seus contribuidores pararem de oferecer suporte.

Sumário da Política de Estabilidade. Contribuidores bem vindos!

Localização

  • Reduzir impacto de incluir icu (footprint binário e de memória).
  • Adotar uma vibrante comunidade de localização.
    • Comunidades de idioma autônomas (30+ registradas até agora)
    • Equipes de evangelismo para aumentar filiação (160+ membros até agora)
    • Melhorar ferramentas para traduzir website, documentação de API, instaladores e conjuntos de apresentações.

Você pode se envolver entrando na comunidade do seu idioma :)
