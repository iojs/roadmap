io.js의 밝은 미래

커뮤니티 주도 로드맵

커뮤니티를 위한, 커뮤니티에 의한

io.js는 Node.js의 대체하며 npm에 있는 npm의 거의 모든 모듈과 호환된다.

io.js는 커뮤니티 주도의 오픈소스 프로젝트이다. io.js는 node.js의 주요 기여자로 시작했지만, 불과 몇 개월 만에 node.js 역사상 가장 많은 수의 활발한 개발자를 모집했다.

io.js에 바라는 것은?

  1. 안정성
  2. 투명성 (디버그, 트레이싱)
  3. 더 좋은 스트림
  4. 장기적인 지원
  5. 지속적인 표준 수용
  6. 지역화

로드맵 프로젝트에서 관심 리스트를 가져왔다. 기여는 환영한다!

안정성 방침

우리는 어떤 언어보다 큰 패키지 생태계를 가지고 있으므로 이를 망쳐서는 안된다.

  • JS API를 제거하지 않는다.
  • 유의적 버전
    • API를 추가할 때는 부 버전을 올린다.
    • V8의 C++ API 변경은 nan에서 관리한다. nan이 처리가능한 경우 부 버전을, 그러지 못할 경우 주 버전을 올린다.

로드맵의 안정성 방침 항목에서 가져왔다. 기여는 환영한다!

채널

Release 버전이 붙은 모든 빌드.
반드시 안정적이고 프로덕션에서 사용가능해야 한다.
Canary Canary V8,다른 새로운 기능을 사용하는 나이틀리 빌드.
NG 차세대(Next Generation) ES6/7 중심적 API.


로드맵에서 가져옴. 기여는 환영한다!

NG (Next Generation)

  • 현대 JavaScript 애플리케이션을 위한 새로운 플랫폼.
  • 계속 하위호환성을 유지할 예정이다.
  • 새 API는 오래된 모듈과 API를 깨지 않으면서 ES6/7 중심적이 될 수 있다.
  • 이후의 API는 현재 플랫폼의 개선과 독립적으로 진행한다.

NG에 관한 토론은 NG 저장소에서 진행중이다.

디버그, 트레이싱

코어는 domains 같은 고 수준 추상화를 목표로 하기보다는 디버그, 트레이스의 생태를 활성화 해야한다.

  • trace_event와의 통합 (V8, Chrome을 위한 새로운 트레이스 API).
  • 디버그 생태를 위한 문서와 교육.
  • 시스템 트레이싱 엔드포인트 통합 (DTrace, LTTng 등)

작업은 트레이싱 워킹 그룹에 의해 주도된다. 기여는 환영한다!

스트림

  • 스트림 워킹 그룹은 readable-stream을 인계해, io.js 공식 stream 모듈로 만들었다.
  • 모든 기존 호환성 이슈 수정.
  • 사용자 에러를 피하기 위해 스트림의 구현 단순화.
  • WHATWG Streams 인터페이스와 호환성 이슈를 명확히 한다.
  • 스트림의 성능향상.

작업은 스트림 워킹 그룹에 의해 주도된다. 기여는 환영한다!

장기적인 지원

  • 새로운 릴리스 라인은 안정성과 의존성 지원과 같이 배포된다.
  • 오래된 버전의 패치 릴리스는 사람들이 적극적으로 기여하는 한 배포된다.
  • io.js는 V8 이전 릴리스 관리자가 지원을 중단할 때 같은 오래된 버전의 의존성을 책임진다.

안정성 방침의 요약이다. 기여는 환영한다!

지역화

  • icu 사용할 때의 영향 감소 (바이너리, 메모리 풋프린트).
  • 활발한 지역화 커뮤니티의 육성.
    • 자율적인 언어 커뮤니티 (30+ 커뮤니티)
    • 에반젤리즘에 의한 멤버 증가 (160+ 맴버)
    • 웹사이트, API 문서, 인스톨러, 슬라이드 덱을 위한 도구 개발.

당신 언어의 커뮤니티에 참여할 수 있다. :)
