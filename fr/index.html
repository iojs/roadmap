L’avenir radieux de io.js

Une feuille de route pilotée par la communauté

Pour la communauté, par la communauté

io.js est une version alternative de node.js et compatible avec la quasi-totalité des modules dans npm.

io.js est un projet open-source dirigé par la communauté. Il a été lancé par les principaux contributeurs de node.js mais dans les quelques premiers mois a attiré plus de développeurs actifs que le projet node.js dans toute son histoire.

Qu'attendent les gens de io.js ?

  1. Stabilité
  2. Transparence (débogage et traçage)
  3. Meilleurs flux (streams)
  4. Support à long terme
  5. Adopter les normes en cours
  6. Localisation

Liste de préoccupations composée à partir du projet feuille de route. Les contributeurs sont les bienvenus !

Politique de stabilité

Nous avons le plus grand écosystème de paquets tous langages confondus, nous ne pouvons pas nous permettre de le briser.

  • Nous ne supprimons pas les API JS.
  • Versionnage sémantique
    • Tout ajout sur l’API signifie une incrémentation mineure de version.
    • Les changements dans l’API C++ de V8 qui peuvent être gérés par nan entraînent une incrémentation mineure de version, et ceux qui ne le peuvent pas entraînent une incrémentation majeure.

Depuis la section de politique de stabilité de la feuille de route. Les contributeurs sont les bienvenus !

Canaux

Release Tout build versionné.
Doit être stable et prêt pour la production.
Canary Nightly builds avec Canary V8 et d’autres nouvelles fonctionnalités.
NG Génération suivante avec des API centrées sur ES6/7.


Depuis la feuille de route. Les contributeurs sont les bienvenus !

NG (Nouvelle Génération)

  • Une nouvelle plate-forme pour les applications JavaScript modernes.
  • Restera rétrocompatible indéfiniment.
  • Les nouvelles API peuvent être centrées sur ES6/7 sans casser les anciens modules et API.
  • Sépare le travail sur les futures API des améliorations de la plateforme actuelle.

Les discussions NG sont en cours sur le dépôt NG.

Débogage & Traçage

Le noyau devrait permettre un écosystème massif et diversifié d’outils de débogage et de traçage plutôt qu’entériner des abstractions de haut niveau comme les domains.

  • S'intégrer avec trace_event (Nouvelle API de traçage pour V8 et Chrome).
  • Documenter et évangéliser un écosystème de débogage.
  • Unifier les points d’accès au système de traçage (DTrace, LTTng, etc.)

Ce travail est piloté par le groupe de travail sur le Traçage. Les contributeurs sont les bienvenus !

Flux (streams)

  • Le groupe de travail sur les Flux a pris la main sur readable-stream, qui est maintenant le module stream officiel pour io.js.
  • Corriger tous les problèmes de compatibilité existants.
  • Simplifier la création de flux pour éviter les erreurs de l’utilisateur.
  • Explorer les interfaces WHATWG Streams pour identifier les problèmes de compatibilité.
  • Améliorer les performances de flux.

Ce travail est piloté par le groupe de travail sur les Flux. Les contributeurs sont les bienvenus !

Support à long terme

  • De nouvelles lignes de release seront livrées avec des dépendances stables et maintenues.
  • Des releases de correctifs pour les anciennes versions seront publiées aussi longtemps que les gens contribuent activement.
  • io.js aura la responsabilité des vieilles versions de dépendances comme V8 qui apparaissent dans les versions précédentes lorsque leurs responsables cessent de les maintenir.

Résumé issu de la politique de stabilité. Les contributeurs sont les bienvenus !

Localisation

  • Réduire l’impact de l’utilisation l’icu (taille du binaire et occupation mémoire).
  • Favoriser les communautés de localisation dynamiques.
    • Des communautés de langue autonomes (plus de 30 enregistrées jusqu’à présent).
    • Efforts d’évangélisation pour développer l’adhésion (plus de 160 membres à ce jour).
    • Améliorer les outils pour traduire le site web, les documentations d’API, les installeurs et les présentations.

Vous pouvez participer en rejoignant la communauté de votre langue :)
