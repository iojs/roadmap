Die Zukunft von io.js

Eine Community-getriebene Roadmap

Für die Community, von der Community

io.js ist ein direkter Ersatz für node.js und ist mit beinahe allen npm Modulen kompatibel.

io.js ist ein von der Community betriebenes Open-Source-Projekt. Es wurde von den Kern-Entwicklern von node.js initiiert, hat aber in den ersten Monaten mehr aktive Entwickler angezogen, als das node.js-Projekt in seiner gesamten Versionsgeschichte.

Was wird von io.js erwartet?

  1. Stabilität
  2. Transparenz (Debugging und Tracing)
  3. Bessere Streams
  4. Langzeitunterstützung
  5. Verwendung aktueller Standards
  6. Lokalisierung

Die Liste der Erwartungen wurde aus dem Roadmap-Projekt zusammengestellt. Mach mit!

Stabilitäts-Richtlinie

Wir verfügen über das größte Paket-Ökosystem unter Programmiersprachen, dabei darf nichts kaputt gehen.

  • Wir entfernen keine JS APIs.
  • Semantic Versioning
    • Jede API-Änderung bedeutet eine kleine Versionsänderung.
    • Änderungen an der C++ API von V8, die durch nan abgedeckt sind, bedeuten einen kleinen Versionssprung und einen großen, wenn sie es nicht sind.

Aus dem Abschnitt "Stabilitäts-Richtlinie" der Roadmap. Mach mit!

Versionszweige

Release Jeder versionierte Build.
Muss für Produktivumgebungen geeignet sein.
Canary Nightly Builds mit Canary-Version von V8
und anderen neuen Features.
NG Next Generation ES6/7-bezogene API.


Auszug aus der Roadmap. Mach mit!

NG (Next Generation)

  • Eine neue Plattform für moderne JavaScript-Applikationen.
  • Wird immer abwärtskompatibel bleiben.
  • Neue APIs können auf ES6/7 fokussiert sein,
    ohne alte Module und APIs zu beeinflussen.
  • Trennt die Arbeit an neuen APIs von der Weiterentwicklung der aktuellen Plattform.

NG-bezogene Diskussionen finden im NG-Repository statt.

Debugging & Tracing

Der io.js-Core soll ein umfangreiches und vielschichtiges Ökosystem von Debugging- und Tracing-Tools ermöglichen, statt die Verwendung von Abstraktionen wie domains zu befürworten.

  • Integration mit trace_event (Neue Trace-API für V8 & Chrome).
  • Dokumentieren und bekannt machen des Debugging-Ökosystems.
  • Gemeinsamen Tracing-Endpunkt schaffen (DTrace, LTTng, etc.)

Unter Leitung der Tracing Working Group. Mach mit!

Streams

  • Die Streams Working Group hat die Arbeit an readable-stream übernommen, das jetzt das offizielle io.js-Stream-Modul ist.
  • Alle existierenden Kompatibilitätsprobleme beheben.
  • Stream-Erzeugung vereinfachen um Benutzerfehler zu vermeiden.
  • WHATWG Streams genauer betrachten und mögliche Kompatibilitätsprobleme identifizieren.
  • Stream-Performance verbessern.

Unter Leitung der Streams Working Group. Mach mit!

Langzeitunterstützung

  • Neue Release-Lines werden mit stabilen und unterstützten Dependencies veröffentlicht.
  • Patch-Releases von älteren Versionen werden so lange veröffentlicht, wie Leute aktiv dazu beitragen.
  • io.js übernimmt die Verantwortung für alte Versionen von Dependencies wie V8, die vor der Veröffentlichung erschienen, wenn deren Entwickler diese nicht mehr unterstützen.

Zusammenfassung der Stability Policy. Mach mit!

Lokalisierung

  • Auswirkungen von icu-Verwendung reduzieren (Binärdatei- und Speichergröße).
  • Fördern einer aktiven Lokalisierungs-Gruppen.
    • Unabhängige Sprach-Communities (bisher über 30 registriert)
    • Bekanntmachung, um weitere Mitglieder zu gewinnen (über 160 bisher)
    • Verbesserung der Tools zur Übersetzung der Website, API-Dokumentationen, Installationsprogrammen und Präsentationen.

Du kannst mitmachen, indem du der Übersetzer-Gruppe für deine Sprache beitrittst. :)
