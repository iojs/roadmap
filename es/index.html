El Futuro Brillante de io.js

Una hoja de ruta dirigida por la comunidad

Para la comunidad, por la comunidad

io.js es un reemplazo intercambiable para node.js y es compatible con casi todos los módulos en npm.

io.js un proyecto de código abierto (open source) dirigido por la comunidad. Fue iniciado por los principales contribuidores de node.js pero en los primeros meses atrajo mas desarrolladores activos que el proyecto node.js ha tenido en toda su historia.

Política de Estabilidad

Tenemos el ecosistema de paquetes más grande de todos los lenguajes, no podemos romperlo.

  • No vamos a eliminar la API de JS.
  • Versionamiento Semántico (SemVer)
    • Cualquier adición a la API significa un incremento de versión menor.
    • Cambios en la API C++ de V8 manejados por nan significan un incremento de versión menor y un incremento de versión mayor si no.

De la sección de Política de Estabilidad de la hoja de ruta. ¡Contribuciones bienvenidas!

Canales

Release Cualquier build versionado.
Debe ser estable y listo para producción.
Canary builds nocturnos con Canary V8 y otras nuevas características.
NG Siguiente generación de API centrada en ES6/7.


De la hoja de ruta. ¡Contribuciones bienvenidas!

NG (Next Generation)

  • Una nueva plataforma para aplicaciones JavaScript modernas.
  • Seguirá siendo compatible con versiones anteriores indefinidamente.
  • Nuevos APIs serán centrados en ES6/7 sin romper viejos módulos y API.
  • Separar el trabajo en API futuras de las mejoras sobre la plataforma actual.

Las discusiones acerca de NG se están haciendo en el repositorio NG.

Depuración y Tracing

core debería habilitar un masivo y diverso ecosistema para herramientas de depuración y tracing mas allá de avalar abstracciones de alto nivel como domains.

  • Integración con trace_event (Nueva API para trace API de V8 y Chrome).
  • Documentar y evangelizar un ecosistema de depuración.
  • Unificar un endpoint para el sistema de tracing (DTrace, LTTng, etc)

Este trabajo está siendo liderado en el Grupo de Trabajo de Tracing. ¡Contribuciones bienvenidas!

Streams

  • Grupo de Trabajo de Streams se ha hecho cargo de readable-stream, el cual es ahora el módulo oficial de stream para io.js.
  • Arreglar todos los problemas de compatibilidad existentes.
  • Simplificar la creación de stream para evitar errores de usuario.
  • Explorar interfaz de WHATWG Streams e identificar problemas de compatibilidad.
  • Mejorar el rendimiento de los stream.

Este trabajo está siendo liderado en el Grupo de Trabajo de Streams. ¡Contribuciones bienvenidas!

Soporte a Largo Plazo (LTS)

  • Nuevas líneas de versiones vendrán con dependencias estables y soportadas.
  • Parches de versiones anteriores serán publicados siempre y cuando existan personas contribuyendo activamente
  • io.js tomará responsabilidad de versiones anteriores de dependencias como V8 que aparezcan en versiones anteriores cuando sus mantenedores dejen de soportarlas.

Resumen de Política de Estabilidad. ¡Contribuciones bienvenidas!

Localización

  • Reducir el impacto de incluir icu (tamaño en binario y memoria).
  • Fomentar una vibrante comunidad de localización.
    • Comunidades autónomas por idioma (30+ registradas hasta ahora)
    • Esfuerzos de evangelización para crecer la afiliación (160+ miembros hasta ahora)
    • Mejorar las herramientas para traducir el sitio web, documentación de la API, instaladores y diapositivas.

Puedes involucrarte, únite a la comunidad de tu idioma :)
