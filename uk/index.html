Для спільноти, спільнотою

io.js є повноцінною заміною node.js та сумісний з майже усіма модулями в npm.

io.js це проект з відкритим вихідним кодом, що керується спільнотою. Він був розпочатий основними учасниками node.js, але в перші кілька місяців залучив більше нових активних учасників, ніж node.js мав за всю історію.

Що саме люди очікують від io.js?

  1. Стабільність
  2. Прозорість (налагодження та відстеження)
  3. Кращі потоки
  4. Довгострокова підтримка
  5. Підтримка поточних стандартів
  6. Локалізація

Список складено з відгуків проекту дорожньої карти. Запрошуємо учасників!

Політика Стабільності

Ми маємо найбільшу екосистему пакетів серед всіх мов, ми не можемо її пошкодити.

  • Ми не видаляємо JS API.
  • Семантичне керування версіями
    • Будь-яке доповнення API означає приріст малої версії .
    • Зміни в V8 в C++ API, які обробляються nan призводять до збільшення малої версії, та збільшення основної, якщо не можуть бути оброблені.

НГ ( Нова Генерація)

  • Нова платформа, для сучасних програм JavaScript.
  • Буде завжди підтримувати зворотню сумісність.
  • Нові інтерфейси можуть бути ES6/7 орієнтованими, не порушуючи старі модулі та API.
  • Відокремити роботу над майбутніми API від поліпшень у поточній платформі.

дискусії стосовно НГ ведуться в окремому НГ проекті.

Відлагодження та Трасування

Ядро має надати велику та різноманітну екосистему інструментів налагодження та трасування замість підтримки абстракцій вищого рівня, як домени.

  • Інтеграція з trace_event (новий API трасування для V8 та Chrome).
  • Документація та євангелізація екосистеми налагодження.
  • Уніфікація системи відстеження кінцевої точки (DTrace, LTTng тощо)

Робота координується Робочою Групою Відстеження. Запрошуємо учасників!

Потоки

  • Робоча Група Потоків взяла на себе відповідальність за readable-stream, який зараз є офіційним io.js stream модулем.
  • Виправити всі існуючи проблеми із сумісністю.
  • Спростити створення потоку, щоб уникнути помилок користувачів.
  • Дослідити інтерфейс WHATWG Потоків і виявити проблеми сумісності.
  • Покращити продуктивність потоків.

Робота координується Робочою Групою Потоків. Запрошуємо учасників!

Довгострокова підтримка

  • Нові релізи будуть поставлятися зі стабільними залежностями, які матимуть довгострокову підтримку.
  • Патч-релізи старих версій будуть опубліковуватися до тих пір, поки люди активно приймають участь в підтримці.
  • io.js візьме на себе відповідальність за підтримку старих версій залежностей, наприклад, V8, які з'являються в попередніх релізах, коли їх підтримка офіційно припиниться.
